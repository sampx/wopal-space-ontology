# 创建或更新 README

**用户输入参数**: `$ARGUMENTS` 

**参数说明**: 项目名。用户未输入时从 `projects/` 目录下项目列表和上下文匹配推断, 有疑问向用户确认。

## 文档路径

- 项目级：`<project>/README.md`
- 若用户偏好语言非英文，先在同目录生成 `README.<locale>.md` 供用户审核；确认通过后，再更新 `README.md`

## 目的

`README.md` 是面向人类读者的项目入口文档。

它应说明：项目是什么、如何安装运行、能做什么、长期参考在哪里。它不是 Agent 规则文件，不是实施 checklist，不是设计规格文档。

## 前置条件

生成前应优先读取：

- 既有项目 `README.md`
- 相关 PRD
- 相关 DESIGN
- 项目 `package.json` / build / run 配置
- 主入口文件或 CLI 命令定义
- License 文件或 package metadata

## 核心规则

- 面向人类读者，使用用户友好的语言。
- 快速开始和核心能力在前，开发命令和参考链接在后。
- 列出模块和核心入口命令，不穷举所有子命令；完整用法以 `--help` 为准。
- 不下沉到内部实现细节；不放置 Agent 专用实施规则。
- 技术栈和项目结构属于 DESIGN / AGENTS.md，README 不重复描述。
- 所有命令必须从 package / config 文件验证，禁止猜测。

## 输出语言与文件命名

- 若用户偏好语言不是英文，必须先生成用户偏好语言版本供用户审核；用户确认通过后，再翻译并更新正式英文版。
- 用户偏好语言版本命名为 `README.<locale>.md`，例如 `README.zh-CN.md`。
- `<locale>` 必须使用 IETF BCP 47 / RFC 5646 标记。
- 正式英文版保持无后缀命名：`README.md`。
- 若用户偏好语言为英文，则直接生成或更新 `README.md`，不生成 `README.en-US.md` 等英文变体文件。

## 写作质量线

- 用一个清晰开头段落说明项目价值。
- 快速开始：列出可执行的安装和运行命令。
- 核心能力：按模块列出功能，不深入实现细节。
- 开发命令：简洁命令表，来源可靠。
- 技术参考：链接长期文档，不重复其内容。
- 用户偏好语言版本的所有标题使用目标语言。

禁止：

- Agent 实施规则
- 深层架构说明
- PRD 路线复述
- 内部任务列表或 TODO
- 未验证命令或猜测的包管理器
- 默认不链接 backlog / task plan / 命令记录等临时产物

## README 模板

```markdown
# <Project Name>

<One-paragraph human-facing description.>

## Quick Start

```bash
<install command>
<run command>
```

## Core Capabilities

| Module | Entry Commands | Purpose |
|---|---|---|

## Development

| Scenario | Command |
|---|---|

## Technical References

| Document | Description |
|---|---|

## License

<license>
```

## 更新模式

更新既有 README 时：

1. 保留准确的用户可见内容。
2. 移除过时命令、过时链接。
3. 补充或修正模块和核心命令列表。
4. 设计和产品细节用链接引用，不复制章节。

## 确认策略

正式写入或覆盖 `README.md` 前，必须先展示完整优化方案并获得用户明确确认。

方案至少包含：

1. 目标文件路径
2. 一句话项目描述
3. 模块 / 核心命令概览
4. 拟新增、修改或删除的部分
5. 将引用的权威文档

未确认前，不得写入、覆盖或重排正式英文版 `README.md`。

若已先生成用户偏好语言版本，用户确认后：

1. 先更新用户偏好语言版本。
2. 再翻译并更新对应英文正式版。
3. 英文正式版必须与已确认版本语义一致。

## 质量检查

- [ ] 目标项目根目录明确或可安全推断
- [ ] README 是项目级，而非目录级
- [ ] 列出模块和核心入口命令，未穷举子命令
- [ ] 安装 / 运行 / 开发命令已从项目文件验证
- [ ] 能力描述面向用户
- [ ] 未重复技术栈或项目结构内容
- [ ] 已链接长期参考文档
- [ ] 没有 Agent 规则内容
- [ ] 若用户偏好语言非英文，已先生成用户偏好语言版本
- [ ] 用户偏好语言版本标题使用目标语言
- [ ] 写入前已展示完整优化方案并获得用户确认
- [ ] 用户确认后已同步更新正式英文版（如适用）

## 完成后响应

用用户语言回复：

1. 更新的文件路径
2. 变更摘要
3. 已验证或缺失的命令
4. 任何假设或缺失的长期参考
