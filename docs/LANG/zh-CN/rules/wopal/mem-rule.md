# 记忆管理规则

## 核心原则

只沉淀与空间优化、项目建设**高度相关**且具有**长期复用价值**的信息。

## 写入前确认规则

无论通过 `memory_manage` 工具或直接编辑 `memory/MEMORY.md`/`memory/diary/`，任何记忆写入操作：

1. **必须先完整展示准备记录的内容**
2. **必须等待用户明确同意**
3. **严禁未经展示和确认直接写入**

适用于：主动沉淀、用户触发（"记下来"）、蒸馏生成。

## 记忆展示格式规范

添加、更新或删除记忆前，必须用以下格式完整展示内容，等待用户确认。展示必须用 **markdown list** 逐字段列出，不得用表格、代码块或纯文本摘要。

### 必须字段

- **分类**：7 个合法分类之一
- **项目**：`wopal-space` 或具体项目名
- **重要性**：0-1 之间浮点数
- **标签**：逗号分隔
- **正文**：完整文本，不得截断、摘要或概括

### 附带信息

- **去重结果**：说明与哪些已有记忆冲突/互补，如无冲突则标注"无重复"
- **工具输出结果**：`memory_manage add/update/delete` 成功后工具返回的完整输出（含 ID、时间等）必须逐字展示给用户

### 示例

```
分类: knowledge
项目: wopal-space-ontology
重要性: 0.95
标签: skill-scripts, ellamaka, restart, hot-reload, verification
正文: 技能脚本变更即时生效，不需要重启 ellamaka...

去重结果：无重复。与 [5397c940] 互补（该记忆描述的是 plugin 需要重启的情况）
```

### 蒸馏预览

`memory_manage distill` 生成的候选记忆列表，每个候选也需按上述格式展示。展示完毕后等待用户选择（全写/精选/取消），确认后才调用 `confirm` 写入。

### 常见错误

- ❌ 只给 ID 或摘要 → 用户看不到完整内容，无法判断是否写入
- ❌ 用表格展示 → 正文过长时表格渲染混乱
- ❌ 工具返回后不展示完整输出 → 用户看不到 ID、时间等元信息

## 不记录

- 通用知识（已知常识、可从文档/网络直接获取）
- 临时状态（版本号、当前配置值）
- 会话流水（做了什么、决策过程）
- 单纯的错误描述（无根因、无方法）

## 分类体系

| 分类 | 标识符 | 默认权重 | 判断标准 |
|------|--------|----------|---------|
| 用户要求 | `requirement` | 0.95 | 用户明确要求必须遵守的规则/行为 |
| 用户档案 | `profile` | 0.90 | 用户个人身份和基本信息 |
| 避坑方法 | `gotcha` | 0.85 | 踩过的坑、不要这样做（反面教训） |
| 经验 | `experience` | 0.85 | 验证有效的最佳实践（正面总结） |
| 偏好 | `preference` | 0.80 | 用户倾向但非强制 |
| 事实 | `fact` | 0.75 | 空间/项目特有的客观事实（通用可查的不记录） |
| 知识 | `knowledge` | 0.70 | 空间/项目特有的技术理解（通用可查的不记录） |

区分要点：`preference` 是"倾向"，`requirement` 是"必须"；`gotcha` 是反面教训，`experience` 是正面实践。

## 写入格式（结论前置六原则）

1. **结论前置**：第一句话就是规则本身，读完即懂。不要用"关键纠正""本次违反""背景："这类元叙述开头，背景和细节放后面
2. **直接描述**：不要 `## [类别]:` 前缀（类别由字段或标题标识）
3. **格式简洁**：普通文本，不要 `**粗体标签**：` 格式
4. **背景必要**：包含"为什么这很重要"或"在什么场景下有用"
5. **内容完整**：保留必要的代码/路径/命令
6. **不截断**：保持完整，去除冗余描述

## 标签设计规范

标签在两个环节起作用：用户搜索时拼入 FTS 查询词；自动注入时通过 conceptBoost 提权。每个标签提交前必须答出"它会命中什么场景的搜索"。

### 选择规则

1. 用检索场景词，不用内容碎片——`委派` 是场景，`Done` 是碎片；`验证纪律` 是场景，`checkbox` 是碎片
2. 英文标签优先（conceptBoost 只对英文词生效，每个查询词匹配 tag 子串 +0.05，上限 +0.15）；中文标签兜底（仅参与 FTS 文本搜索，无 boost）
3. 禁止拼接造词（`context-manage-detail` 没人搜），禁止宽泛词（"操作""处理"）
4. 标签数量 ≤5 个——超了说明正文焦点不清晰

### 写入前自检

- 每个标签能说出一条真实检索场景（"搜什么会命中这个标签"）
- 英文标签覆盖所有核心概念（是 boost 主力通道）
- 无内容碎片、无拼接长词、无宽泛词

## 沉淀位置

| 信息性质 | 存储位置 |
|----------|----------|
| 长期知识/经验/避坑 | LanceDB（`memory_manage`），不可用时降级到 `.wopal-space/memory/MEMORY.md` |
| 用户偏好 | `.wopal-space/memory/USER.md` |
| 空间工作守则/约定 | `.wopal-space/REGULATIONS.md` |
| 项目或模块特定规则 | 项目根或本目录下 `AGENTS.md` |
| 核心行为特质 | 灵魂提示词 |
| 短期会话记忆 | `.wopal-space/memory/diary/` |

## 沉淀决策流程

### 触发时机

- 用户说"记下来/记录知识" → 提取关键事实写入
- 发现重复出现的信息 → 主动提醒沉淀价值，等确认后执行
- 用户偏好含"应该/必须" → 提炼到 `requirement`

### 逐条筛选

每条候选依次通过以下判断，未通过的在展示时标注原因：

1. **重复检查**：与已有记忆重复 → 跳过或合并
2. **通用性过滤**：属于通用可查知识（见"不记录"）→ 跳过
3. **时效性判断**：已修复/已完成问题 → 跳过
4. **会话流水过滤**：属于会话流水（见"不记录"）→ 跳过
5. **价值检验**：未来 Agent 读到能避免错误或做出更好决策吗？不能则跳过
6. **格式合规**：符合结论前置六原则，不合规则修正
